# Plan: Native Word Cross-References (Todo #17)

## Context

Cross-references in Lexicon Markdown (`[clause X](#anchor)`) currently render as **plain text** in the .docx output (e.g. "Clause 1.2"). This means no clickable navigation and no visual link indication. We want to convert these into Word-native internal hyperlinks so that Ctrl+click in Word jumps to the referenced clause.

**Note on auto-updating**: The todo mentions "allow cross-references to update automatically when clause numbers change in Word". This is **not fully achievable** because clause numbers come from Word's native numbering engine — a REF field can't capture a number generated by the numbering engine. What *is* achievable is clickable navigation via bookmarks + hyperlinks, which is the primary value.

## Feasibility: Confirmed

docx-rs 0.4.19 has full support for both pieces:

1. **Bookmarks**: `Paragraph::add_bookmark_start(id, name)` / `add_bookmark_end(id)` — places `<w:bookmarkStart>` / `<w:bookmarkEnd>` in the XML
2. **Internal hyperlinks**: `Hyperlink::new(bookmark_name, HyperlinkType::Anchor).add_run(run)` — generates `<w:hyperlink w:anchor="name">` which gives Ctrl+click navigation in Word

No field codes or ZIP post-processing needed.

## Implementation Plan

### 1. Bookmark name sanitiser (`render/common.rs`)

Word bookmark names: must start with a letter, only `[A-Za-z0-9_]`, max 40 chars.

```rust
fn bookmark_name(anchor_id: &str) -> String
```

- Replace `-` with `_`
- Prefix with `lx_` (avoids collision with Word's reserved `_` prefix bookmarks)
- Truncate to 40 chars

### 2. Bookmark map setup (`render/docx.rs`)

At the start of `render_docx()`, build a `HashMap<String, usize>` mapping each `anchor_id` → a unique bookmark integer ID. Iterate over the existing `anchor_map` keys and assign incrementing IDs (starting at 1).

Pass this map through to clause/addendum rendering functions.

### 3. Place bookmarks at anchor targets (`render/docx.rs`)

In `render_clause()` (~line 289): when `clause.anchor` is `Some(id)`, wrap the heading paragraph (or first content paragraph for non-headed clauses) with bookmark start/end:

```rust
if let Some(ref anchor_id) = clause.anchor {
    if let Some(&bm_id) = bookmark_map.get(anchor_id) {
        let name = bookmark_name(anchor_id);
        para = para.add_bookmark_start(bm_id, &name)
                   .add_bookmark_end(bm_id);
    }
}
```

Similarly for addendum heading paragraphs (where `addendum.anchor` is set), and recital clauses.

### 4. Render cross-refs as hyperlinks (`render/common.rs`)

Change `add_inline_run()` for the `CrossRef` variant. Instead of:
```rust
para.add_run(run)  // plain text
```

Use:
```rust
let link = Hyperlink::new(bookmark_name(anchor_id), HyperlinkType::Anchor)
    .add_run(run);
para.add_hyperlink(link)
```

The `anchor_id` is already available on `InlineContent::CrossRef { anchor_id, .. }` — currently ignored with `..`.

### 5. Files to modify

| File | Change |
|------|--------|
| `src/render/common.rs` | Add `bookmark_name()` fn; change `CrossRef` arm in `add_inline_run()` to emit `Hyperlink` instead of plain `Run` |
| `src/render/docx.rs` | Build bookmark map; pass it through rendering; add `bookmark_start`/`bookmark_end` on anchored paragraphs |

### 6. Imports needed

In `render/common.rs`: add `Hyperlink`, `HyperlinkType` from `docx_rs`.
In `render/docx.rs`: no new imports needed (bookmark methods are on `Paragraph`).
